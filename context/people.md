## Decision Authority

| Decision Type | Who Decides |
|---------------|-------------|
| Scope (what is in/out of v1) | PRD v1.0 is authoritative. If not in the PRD, it goes in the backlog. |

| Model changes | Builder, but must log reasoning (cost, quality tradeoff) |
| Stack deviations | Builder, but must document in `/context/decisions.md` before deviating |

- DNS / domain access for uslsystems.co: whoever controls the domain registrar
- OpenRouter billing or model availability: OpenRouter support

# v1.1 Improvements Plan

Created: 2025-04-09

---

## Wave 1 — Quick Fixes (do now)

### 1.1 Fix Quality Checker Scoring
Quality score returns ~30/100 on good summaries. Root cause likely: regex heading matching fails on commas/special chars, reports sections as "missing" when they exist. Fix the regex, recalibrate scoring formula.

### 1.2 Better Document Naming
Use meeting title from metadata if provided: `acme-q3-review-2025-04-09.md`. Fallback to meeting type: `sales-discovery-2025-04-09.md`. Sanitize for filenames. Apply to .md and .docx exports.

### 1.3 "New Summary" Reset Button
After export, let users clear the form and start fresh. Button below the export toolbar or in the header.

### 1.4 Scroll to Summary
After generation completes, auto-scroll to the summary area. Users with long transcripts don't see the result otherwise.

---

## Wave 2 — Core Quality (do next)

### 2.1 Adaptive Caps Based on Transcript Length
Replace hard caps with dynamic limits based on chunk count (proxy for meeting duration):

| Transcript Length | Exec Summary | Decisions | Quotes | Next Steps |
|-------------------|-------------|-----------|--------|------------|
| < 15 min (~1-2 chunks) | 3 bullets | 3 max | 3 | 5 rows |
| 15-45 min (~3-6 chunks) | 5 bullets | 5 max | 4 | 7 rows |
| 45-90 min (~7-12 chunks) | 7 bullets | 7 max | 6 | 10 rows |
| > 90 min (~13+ chunks) | 8 bullets | 8 max | 8 | 12 rows |

Inject caps into reducer prompt dynamically based on chunk count.

### 2.2 Separate Follow-Up Email Draft
Extract email section from summary markdown. Display in a dedicated tab or panel next to the summary, not buried at the bottom. Prominent "Copy Email" action. Pre-formatted for paste into Outlook/Teams.

### 2.3 Extract page.tsx Into Components
Split ~400-line page.tsx into: `InputForm`, `SummaryDisplay`, `ExportToolbar`, `QuotePanel`, `EmailDraftPanel`. Improves maintainability and makes Wave 3 features easier to add.

### 2.4 Prompt Tuning Per Meeting Type
Test against 5+ real transcripts per type. Adjust emphasis weights. Sales Discovery should surface objections aggressively. Internal Sync should prioritize owners on every action item. Customer Support should flag escalation risk.

---

## Wave 3 — UX Polish

### 3.1 Drag-and-Drop File Upload
Add drop zone on the transcript textarea. Visual feedback on drag-over. Same file parsing logic already built.

### 3.2 Toast Notifications
Replace static inline error/success divs with toast notifications for: clipboard copy, file download, errors, warnings. Use a lightweight toast library or build with Tailwind.

### 3.3 Streaming Progress Messages
Replace "Processing chunk 2 of 7..." with descriptive stages: "Cleaning transcript...", "Analyzing decisions...", "Extracting quotes...", "Writing summary...". Gives users confidence the tool is working.

### 3.4 Editable Summary
Let users fix a quote, add a missing action item, or tweak the email draft before exporting. Contenteditable or textarea toggle on the summary area. Edits persist to the export.

### 3.5 Meeting Type Auto-Detection
Analyze first 500 chars of transcript. If speaker says "demo", "pricing", "proposal" → suggest Sales Discovery. If "ticket", "issue", "resolved" → suggest Customer Support. Light heuristic, not a model call.

### 3.6 Summary History in localStorage
Save last 10 summaries in browser storage. Simple history dropdown to revisit past summaries without regenerating. No database needed.

---

## Wave 4 — Automation & Integration

### 4.1 Auto-Send via Power Automate Webhook
- "Send to Teams" button in export toolbar
- POSTs structured JSON (title, date, type, markdown, action items) to a configured webhook URL
- Webhook URL stored in env var `POWERAUTOMATE_WEBHOOK_URL`
- Power Automate flow handles routing to Teams channel, OneDrive, SharePoint
- No OAuth, no Azure AD app, no token management in our app
- Users customize the Power Automate flow independently

### 4.2 Teams Chat Log (Future)
Keep the optional chat log textarea in the UI but collapsed. Test on 3 real meetings with chat logs to determine if it materially improves output. If yes, consider separate chunking for chat. If no, simplify to a checkbox: "Include chat context: yes/no" with a simpler prompt injection.
